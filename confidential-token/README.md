# Soroban workspace — confidential-token

Cargo workspace containing the on-chain side of the Confidential Token
project. See [../README.md](../README.md) for the full picture and
[../docs/architecture.md](../docs/architecture.md) for the design.

## Crates

| Path                                      | Type          | Purpose                                    |
|-------------------------------------------|---------------|--------------------------------------------|
| `crates/baby-jubjub`                      | library       | BabyJubJub Edwards point arithmetic        |
| `crates/eerc-types`                       | library       | Shared `#[contracttype]` structs           |
| `contracts/verifier-registration`         | contract      | Groth16 verifier, 5 public signals         |
| `contracts/verifier-transfer`             | contract      | Groth16 verifier, 32 public signals        |
| `contracts/verifier-withdraw`             | contract      | Groth16 verifier, 16 public signals        |
| `contracts/registrar`                     | contract      | BabyJubJub PK registry                     |
| `contracts/confidential-token`            | contract      | Main orchestrator                          |

## Quick start

```bash
cargo test -p baby-jubjub                       # native unit tests
cargo build --target wasm32v1-none --release    # all five WASM
ls -lh target/wasm32v1-none/release/*.wasm
```

## Regenerating verifier WASM bindings

The verifier contracts embed verification key bytes from
`EncryptedERC/circom/build/<circuit>/verification_key.json`. They were
generated by `soroban-verifier-gen` v0.2:

```bash
cargo install soroban-verifier-gen --version 0.2.0
soroban-verifier-gen \
  --vk EncryptedERC/circom/build/registration/registration_verification_key.json \
  --out contracts/verifier-registration \
  --name RegistrationVerifier
# repeat for transfer and withdraw
```

After regeneration, run `cargo build --target wasm32v1-none --release` and
verify each WASM is < 64 KB.
